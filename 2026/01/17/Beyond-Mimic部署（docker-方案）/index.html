Beyond Mimic部署（docker 方案）

Isaac-Sim Isaac-Lab BeyondMimic Robotics Docker
Robotics
发布日期:   2026-01-17

1 前置准备

  • Linux环境，能装docker就行
  • 安装docker
  • 处理好自己的网络环境

    1.1 docker安装

    自己装吧

    1.2 配置nvidia穿透

    自己穿透吧

2 安装isaac-lab

beyond mimic官方要求的安装版本是lab2.1+sim4.5的版本。如果lab是2.3，在训练的时候会报错缺少模块。

2.1 isaac-sim安装

安装isaac-sim 4.5版本

docker pull nvcr.io/nvidia/isaac-sim:4.5.0

安装isaac-sim 5.0版本

docker pull nvcr.io/nvidia/isaac-sim:5.0.0

nvcr.io的这个源貌似是不需要开代理下载的，因为网络问题报错，可以按照以下顺序进行尝试下载：

  1. 开Tun模式
  2. 开系统代理模式
  3. 单独配置docker走代理

    2.2 isaac-lab安装

    使用git下载源码
    # 随便给自己创建一个工程文件夹
mkdir -p ~/workspace
cd ~/workspace

# 克隆仓库
git clone https://github.com/isaac-sim/IsaacLab.git

# 切换到你需要的版本 2.1.0 (如果需要最新版则跳过此步)
cd IsaacLab
git checkout v2.1.0
    也可以手动在github网站上下载对应版本的压缩包然后解压
    unzip IsaacLab-2.1.0.zip
    给docker使用本地图像的权限
    xhost +local:root
    进去isaac-sim的docker容器，开始安装isaac-lab2.1，Sim4.5
    docker run --name isaac-lab --entrypoint bash -it --gpus all --rm --network=host \
    -e "ACCEPT_EULA=Y" \
    -e "PRIVACY_CONSENT=Y" \
	-p 6006:6006 \
    -p 49152:49152 \
    -v /home/robot/cjy/IsaacLab-2.2.0-Sim-5.0:/workspace/IsaacLab \
    -v /tmp/.X11-unix:/tmp/.X11-unix \
    -e DISPLAY=$DISPLAY \
    -v $HOME/.Xauthority:/root/.Xauthority \
    nvcr.io/nvidia/isaac-sim:5.0.0
    进去isaac-sim的docker容器，开始安装isaac-lab2.2，Sim5.0(貌似使用这个版本，在安装lab的时候会损害pip，导致需要重装pip)
    docker run --name isaac-lab --entrypoint bash -it --gpus all --rm --network=host \
    -e "ACCEPT_EULA=Y" \
    -e "PRIVACY_CONSENT=Y" \
	-p 6006:6006 \
    -p 49152:49152 \
    -v /home/robot/cjy/IsaacLab-2.2.0-Sim-5.0:/workspace/IsaacLab \
    -v /tmp/.X11-unix:/tmp/.X11-unix \
    -e DISPLAY=$DISPLAY \
    -v $HOME/.Xauthority:/root/.Xauthority \
    nvcr.io/nvidia/isaac-sim:5.0.0
    开始安装之前先配置一下命令行代理
    export HTTP_PROXY="http://192.168.1.124:7897"
export HTTPS_PROXY="http://192.168.1.124:7897"
    开始安装isaac-lab
    # 1. 进入挂载的目录
cd /workspace/IsaacLab

# 2. 创建软链接 (Isaac Sim 在容器里的位置通常是 /isaac-sim)
# 注意：v2.1.0 版本可能脚本会自动处理，但手动做一下保险
ln -s /isaac-sim _isaac_sim

# 3. 开始编译和安装
# 这会安装 pytorch 2.5.1 等依赖，pip 会自动处理
apt update
apt install git -y
apt install cmake build-essential -y

./isaaclab.sh --install
./isaaclab.sh -p -m pip install scipy==1.10.1 "numpy<2.0"
    测试是否安装成功
    # 检查 Python 环境
/isaac-sim/python.sh -c "import isaaclab; import torch; print(f'IsaacLab loaded. CUDA: {torch.cuda.is_available()}')"
    测试图形化界面是否能正常拉起
    ./isaaclab.sh -p scripts/demos/bipeds.py

    3 Beyond Mimic安装

    3.1 下载源码

    下载源码到isaac的同级目录。注意！不是下载在IsaacLab目录里面
    cd /workspace
git clone https://github.com/HybridRobotics/whole_body_tracking.git
    从GCS下载机器人描述文件
    # Enter the repository
cd whole_body_tracking
# Rename all occurrences of whole_body_tracking (in files/directories) to your_fancy_extension_name
curl -L -o unitree_description.tar.gz https://storage.googleapis.com/qiayuanl_robot_descriptions/unitree_description.tar.gz && tar -xzf unitree_description.tar.gz -C source/whole_body_tracking/whole_body_tracking/assets/ && rm unitree_description.tar.gz
    安装这个库
    /isaac-sim/python.sh -m pip install -e source/whole_body_tracking

    3.2 验证Beyond Mimic安装

    Beyond Mimic做的工作是将已经重定向好的机器人关节动作序列（csv）转成NPZ用去强化学习学习这个动作，为了管理海量的要训练的动作，该仓库会自动将转化好的NPZ存储在Wandb云端，训练时候会从云端load这个NPZ的数据。

    3.2.1 准备要训练的关节动作序列

    这个可以自己去用动作捕捉设备去采集然后重定向到要跑的机器人获得这个csv数据，也可以下载别人已经重映射好的csv动作序列。下面是官方仓库给的一些下载东西。
  • Unitree-retargeted LAFAN1 Dataset is available on HuggingFace
  • Sidekicks are from KungfuBot
  • Christiano Ronaldo celebration is from ASAP.
  • Balance motions are from HuB
    这里以训练宇树的g1机器人dance1_subject1.csv动作为案例，该文件的位置在下面的链接lvhaidong/LAFAN1_Retargeting_Dataset at main。

    3.2.2 配置W&B Registry

    自己注册一个账户Weights & Biases: The AI developer platform
    在左侧栏目录下的Core下面的Registry，创建一个新的Registry，Name是Motions，Accepted artifact types设置为All types。

    3.2.3 执行转换并自动上传

    官方参考命令
    python scripts/csv_to_npz.py --input_file {motion_name}.csv --input_fps 30 --output_name {motion_name} --headless
    实际执行命令
    /isaac-sim/python.sh scripts/csv_to_npz.py --input_file dance1_subject1.csv --input_fps 30 --output_name dance1_subject1 --headless
    第一次执行命令时，选择2使用已有的账户，然后点击链接创建自己的api key，按照要求复制自己的api key过去，复制的时候是像密码那样不会显示的。
    当命令行显示[INFO]: Motion saved to wandb registry: motions/dance1_subject1表示已经上传成功，直接ctrl+c关掉命令行就行了

    3.2.4 查看一下重定向的动作

    可以从云端拉取刚刚上传的动作文件，在isaac-lab图形化看一下。
    官方参考命令
    python scripts/replay_npz.py --registry_name={your-organization}-org/wandb-registry-motions/{motion_name}
    实际运行命令
    /isaac-sim/python.sh scripts/replay_npz.py --registry_name=demochangechoice-hong-kong-university-of-science-and-tec-org/wandb-registry-Motions/dance1_subject1
    可以看到机器人跳舞的重映射动作。

3.2.5 训练机器人

官方参考命令

python scripts/rsl_rl/train.py --task=Tracking-Flat-G1-v0 \
--registry_name {your-organization}-org/wandb-registry-motions/{motion_name} \
--headless --logger wandb --log_project_name {project_name} --run_name {run_name}

实际执行命令

/isaac-sim/python.sh scripts/rsl_rl/train.py --task=Tracking-Flat-G1-v0 --registry_name demochangechoice-hong-kong-university-of-science-and-tec-org/wandb-registry-Motions/dance1_subject1 --headless --logger wandb --log_project_name dance_project --run_name test_dance

监控训练过程

/isaac-sim/python.sh -m tensorboard.main --logdir=. --bind_all

3.2.6 查看机器人的训练效果

官方参考命令

python scripts/rsl_rl/play.py --task=Tracking-Flat-G1-v0 --num_envs=2 --wandb_path={wandb-run-path}

实际执行命令

/isaac-sim/python.sh scripts/rsl_rl/play.py --task=Tracking-Flat-G1-v0 --num_envs=2 --wandb_path=demochangechoice-hong-kong-university-of-science-and-tec/dance_project/s7rall9c

4 打包固化docker

4.1 将当前容器打包成镜像

sim4.5，lab2.1版本

docker commit isaac-lab sim4.5-isaac-lab:2.1.0

sim5.0，lab2.2版本

docker commit isaac-lab sim5.0-isaac-lab:2.2.0

4.2 从镜像创建容器

sim4.5，lab2.1版本

docker run --name isaac-ready -it --gpus all --rm \
    -e "ACCEPT_EULA=Y" \
    -e "PRIVACY_CONSENT=Y" \
	-p 6006:6006 \
    -p 49152:49152 \
	-v /home/robot/cjy/IsaacLab-release-2.1.0:/workspace/IsaacLab \
    -v /tmp/.X11-unix:/tmp/.X11-unix \
    -e DISPLAY=$DISPLAY \
    -v $HOME/.Xauthority:/root/.Xauthority \
    sim4.5-isaac-lab:2.1.0

sim5.0，lab2.2版本

docker run --name isaac-ready -it --gpus all --rm \
    -e "ACCEPT_EULA=Y" \
    -e "PRIVACY_CONSENT=Y" \
	-p 6006:6006 \
    -p 49152:49152 \
	-v /home/robot/cjy/IsaacLab-release-2.2.0:/workspace/IsaacLab \
    -v /tmp/.X11-unix:/tmp/.X11-unix \
    -e DISPLAY=$DISPLAY \
    -v $HOME/.Xauthority:/root/.Xauthority \
    sim5.0-isaac-lab:2.2.0
