配置github ssh密钥

1 前言

每次配置GitHub的SSH密钥都忘记命令，每次都要问GPT，这里写个博客记录一下。

2 开始配置

2.1 设置用户名

git config --global user.name "JunyongCHEN"

这里的 “Your Name” 建议使用你的 GitHub 用户名。

2.2 设置邮箱

git config --global user.email "your_email@example.com"

2.3 生成 SSH 密钥对

首先检查你的系统是否已经有密钥，如果有（比如看到 id_rsa.pub 或 id_ed25519.pub），可以跳过生成步骤。

在终端输入：

ls -al ~/.ssh

如果没有，运行以下命令生成新密钥（推荐使用更安全的 ed25519 算法）：

ssh-keygen -t ed25519 -C "你的GitHub邮箱@example.com"
  • **提示 Enter file…**：直接按 回车 保持默认路径。
  • **提示 Enter passphrase…**：设置一个密钥密码（可选）。如果不想每次提交都输密码，直接连按两次 回车 跳过。

2.4 将 SSH 密钥添加到 GitHub

你需要把生成的公钥内容复制到 GitHub 网站上。

获取公钥内容

cat ~/.ssh/id_ed25519.pub

你会看到一串以 ssh-ed25519 开头的长字符串，完整复制它。
