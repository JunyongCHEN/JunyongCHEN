RBCC 视觉引导粉末涂装缺陷检测与自动补喷系统

1 项目背景

这个项目是我在大四参加的港科广举办的红鸟挑战营（RBCC）中，与团队成员共同完成的一个粉末涂装产线智能化改造原型。项目聚焦于传统粉末涂装生产线中普遍存在的人工补喷依赖度高、返工率大、劳动环境恶劣等痛点，尝试用视觉算法 + 边缘 NPU + 运动控制的方式，构建一套低成本、可落地的自动补喷原型。

1.1 行业痛点

在 2025 年 6 月的红鸟挑战营企业调研中，我们走访了广州泽亨、广东信源、西江鸿盛环保、明珠电气四家企业，覆盖喷涂制造、物流设备、环保回收、电力设备四个细分领域。最终选择以 广东信源物流设备有限公司 的粉末涂装产线为主要场景，原因是其痛点最集中、AI 技术适配度最高、商业价值最清晰。

信源涂装组的关键数据：

指标 现状
日产能 300 挂，满负荷 660 挂
每周工件种类 100–200 种不同形状
人工补喷率 100%（所有工件均需人工补喷）
常规工件补喷面积 ~10%
复杂工件补喷面积 20%–30%
单件补喷时间 2–5 分钟
专业补喷工数量 3 人
新员工培训周期 3 个月（部分需半年）

由此归纳出的核心痛点：

  • 100% 依赖人工补喷：自动粉房喷枪只能上下左右移动，无法转弯，折弯区、挂钩接触位、工件两端等形成大量喷涂盲区；
  • 产线柔性差：每周 100–200 种非标工件，缺乏标准模板，固定轨迹自动化设备难以通用；
  • 技能人才稀缺：补喷工培训周期长达 3 个月，技能依赖经验，难以快速复制；
  • 返工浪费严重：喷涂不均、漏喷、堆粉导致返工率长期排名全产线前五，粉末涂料与表面处理材料浪费大；
  • 工作环境恶劣：粉尘浓度高、气味刺激，工人需穿戴厚重防护服、防毒面具，夏季高温下作业体验差。

在四家企业综合评分中，信源以 8.7 分（满分 10 分）位居第一，显著高于泽亨（6.1）、西江鸿盛（7.9）和明珠电气（6.2），因此我们将其作为原型验证的目标场景。

1.2 核心思路

我们的目标不是一次性替换整条产线，而是先做一套可验证的闭环原型：

  1. 用工业相机采集工件图像；
  2. 用 YOLOv11 检测漏喷 / 缺陷区域；
  3. 对缺陷边缘像素坐标做标定，映射为 X-Y-Z 三轴步进电机的运动步数；
  4. 驱动二自由度补喷头完成定点补喷；
  5. 整套算法部署在 RK3588 NPU 上，实现端侧实时推理。

3 系统架构

整个系统从功能上划分为四层：

层级 功能
UI 交互模块 缺陷识别展示、运动路径展示、手动控制
视觉算法模块 内角区域识别、漏喷区域识别（YOLO）、NPU 加速、运动路径计算
控制功能模块 机械臂舵机控制、流水线电机控制、激光限位
机械硬件设计 二维机械臂、二自由度喷枪、模拟流水线

各模块的详细设计与分工：

  • 机械硬件设计：由团队中负责结构与硬件的杨同学设计，采用舵机驱动的二自由度机械臂控制喷枪上下与前后移动，由电机控制流水线吊机带动工件左右移动；末端喷枪同样由两自由度舵机控制，能够覆盖传统固定喷枪无法到达的死角区域。
  • 控制功能模块：我负责与算法模块的接口对接，解析视觉算法输出的缺陷掩膜，将路径计算结果转化为时序上的舵机与电机控制参数；同时通过激光限位保证工件在视觉识别区与自动补喷区的位置一致。
  • 视觉算法模块：团队里负责算法的李同学和高完成，结合 YOLO 与 OpenCV 图像处理技术。YOLO 负责识别大面积漏喷区域和焊接点等重点区域；OpenCV 通过边缘检测、形态学梯度与局部增强提取内角和复杂结构区域，最终融合生成缺陷掩膜。
  • 硬件加速模块：由我负责，将训练好的 YOLO 模型进行量化裁剪，部署到 RK3588 嵌入式开发板，利用其 NPU 能力对推理进行加速。
  • UI 交互模块：负责前端的李同学开发草稿，我负责真机部署和完善，实时展示工件识别图像、自动标注缺陷区域、动态模拟机械臂补喷路径，并支持手动控制模式以处理特殊工件。

从产线流程看，系统被布置在三个连续工位：

  1. 自动涂装区：模拟现有自动喷粉工位；
  2. 漏喷检测区：工业相机 + 激光限位，视觉算法识别漏喷区域；
  3. 自动补喷区：机械臂根据检测结果执行补喷路径；
  4. 后续衔接质检区与加热固化区。

4 我的职责：端侧部署与运动控制

我在团队中主要负责 RK3588 端侧部署 与 运动控制闭环 两部分：

  • 模型转换与量化：将 YOLOv11n 从 PyTorch 导出为 ONNX，再通过 RKNN-Toolkit2 转换为 RK3588 NPU 可用的 .rknn 格式，并进行 INT8 量化；
  • 端侧推理工程：在 RK3588 上部署推理脚本，打通图像采集 → 预处理 → NPU 推理 → 后处理 → 结果输出的流程；
  • 坐标映射：将检测框 / 边缘像素坐标通过相机标定映射为电机步数；
  • 运动控制：基于自定义 PCB 扩展板，驱动 X-Y-Z 三轴步进电机与喷枪舵机，实现定点补喷；
  • 系统联调：与算法、结构、UI 同学对接，完成端到端闭环验证。

5 YOLOv11 缺陷检测

5.1 数据集与训练

漏喷检测被抽象为一个目标检测任务。团队里负责算法的李同学使用橙色 3D 打印工件作为实验对象，针对两类典型缺陷进行标注：

  • 漏喷区域（Missed Spray）
  • 内角 / 沟槽区域（Inner Groove）

训练流程：在 PC 端用 YOLOv11n 做迁移学习，降低参数量以适应边缘部署；训练完成后在测试集上验证检测框的置信度和定位精度。最终我拿到训练好的 .pt 权重，负责后续的 ONNX 导出与 RK3588 部署。

5.2 内角沟槽检测

对于复杂工件的内角、沟槽等几何特征，高同学使用传统的图像处理做边缘轮廓提取：

流程包括 Mask 生成、边缘检测、深度测量与形状检测，最终得到缺陷区域的轮廓顶点、交点和内角轮廓，用于后续路径规划。

6 RK3588 NPU 部署与量化

6.1 为什么选 RK3588

RK3588 的 NPU 算力为 6 TOPS，在边缘端属于性价比很高的选择。我们的需求是：

  • 640×640 输入的 YOLOv11n 实时推理；
  • 低功耗、小体积，便于集成到产线控制柜；
  • 有成熟的 RKNN 工具链，支持 ONNX → RKNN 转换和 INT8 量化。

6.2 ONNX → RKNN 转换

模型转换脚本如下（基于 rknn-toolkit2）：

import sys
from rknn.api import RKNN

DATASET_PATH = '../../../datasets/COCO/coco_subset_20.txt'
DEFAULT_RKNN_PATH = '../model/yolo11.rknn'
DEFAULT_QUANT = True

def parse_arg():
    if len(sys.argv) < 3:
        print("Usage: python3 {} onnx_model_path platform [dtype] [output_path]".format(sys.argv[0]))
        print("       platform: rk3562/rk3566/rk3568/rk3588/rk3576")
        print("       dtype: i8/u8/fp")
        exit(1)
    model_path = sys.argv[1]
    platform = sys.argv[2]

    do_quant = DEFAULT_QUANT
    if len(sys.argv) > 3:
        model_type = sys.argv[3]
        if model_type not in ['i8', 'u8', 'fp']:
            print("ERROR: Invalid model type: {}".format(model_type))
            exit(1)
        elif model_type in ['i8', 'u8']:
            do_quant = True
        else:
            do_quant = False

    output_path = sys.argv[4] if len(sys.argv) > 4 else DEFAULT_RKNN_PATH
    return model_path, platform, do_quant, output_path

if __name__ == '__main__':
    model_path, platform, do_quant, output_path = parse_arg()

    rknn = RKNN(verbose=False)

    print('--> Config model')
    rknn.config(mean_values=[[0, 0, 0]], std_values=[[255, 255, 255]], target_platform=platform)
    print('done')

    print('--> Loading model')
    ret = rknn.load_onnx(model=model_path)
    if ret != 0:
        print('Load model failed!')
        exit(ret)
    print('done')

    print('--> Building model')
    ret = rknn.build(do_quantization=do_quant, dataset=DATASET_PATH)
    if ret != 0:
        print('Build model failed!')
        exit(ret)
    print('done')

    print('--> Export rknn model')
    ret = rknn.export_rknn(output_path)
    if ret != 0:
        print('Export rknn model failed!')
        exit(ret)
    print('done')

    rknn.release()

关键参数说明：

  • mean_values=[[0,0,0]] 与 std_values=[[255,255,255]]：将输入归一化到 [0, 1]；
  • do_quantization=True：启用 INT8 量化，显著降低模型体积和推理延迟；
  • dataset：量化校准数据集，建议覆盖实际产线中常见的工件类型。

6.3 端侧推理

转换后的 .rknn 模型在 RK3588 上通过 librknn_api 或 Python API 加载。端侧推理脚本主要完成：

  1. 从摄像头读取帧；
  2. 做 letterbox 缩放与归一化；
  3. 调用 NPU 推理；
  4. 后处理：置信度过滤 + NMS，得到缺陷框；
  5. 提取缺陷框边缘像素坐标，送入运动控制模块。

实际部署中需要特别注意 NPU 后处理与原始 YOLO 输出头的对齐，RK3588 上我们用的是 rknn_model_zoo 中 YOLOv5/YOLOv8 的后处理范式做适配。

7 坐标映射与运动控制

这是整个系统中最关键、也是最容易出问题的环节：

像素坐标 → 物理坐标 → 电机步数

7.1 坐标映射

相机固定安装后，通过标定获得像素平面与机械臂工作平面的单应性关系。对检测到的缺陷区域，我们采样其边缘像素坐标，再映射为 X-Y 轴的物理位置；Z 轴用于控制喷头与工件表面的距离。

需要注意的几点：

  • 相机畸变校正：工业相机畸变较小，但如果使用广角镜头需要先做 cv2.undistort；
  • 流水线运动补偿：工件在传送带上移动时，检测坐标需要结合编码器反馈做时间同步；
  • 喷头延迟：电磁阀开启到出粉有几十毫秒延迟，需要提前触发。

7.2 运动控制硬件

我为 RK3588 设计了一块自定义扩展板，用于驱动步进电机、舵机和激光限位模块，我自己完成了焊接工作：

板子主要资源：

  • 多路步进电机驱动接口（X / Y / Z 三轴）；
  • 舵机控制接口（喷枪俯仰 / 开关）；
  • 激光限位输入；
  • 与 RK3588 GPIO / I2C / UART 对接。

7.3 控制流程

  1. 视觉模块输出缺陷像素坐标；
  2. 主控程序将坐标换算为电机目标步数；
  3. 通过梯形加减速曲线驱动步进电机到目标位置；
  4. 到达后打开喷枪电磁阀，完成局部补喷；
  5. 关闭喷枪，复位或移动到下一个缺陷点。

我们在实验室环境下完成了完整闭环：

  1. 将待补喷工件放到模拟传送带上；
  2. 相机捕获图像，RK3588 NPU 实时检测缺陷；
  3. 坐标映射后，机械臂带动喷枪移动到缺陷上方；
  4. 喷枪完成定点补喷；
  5. 工件进入下一工位做质检。

原型验证了「视觉检测 → 坐标映射 → 执行器控制」这条技术路径的可行性，但在精度、节拍、鲁棒性上与工业落地还有距离。

按报告中的估算：

  • 总投资：约 200 万元（设备 150 万 + 软件 50 万）
  • 年收益：约 245 万元
    • 人工成本节约：60 万/年（减少 4 名工人）
    • 材料损耗节约：20 万/年
    • 质量成本节约：15 万/年
    • 产能提升收益：100 万/年
    • 质量溢价收益：50 万/年

  • 投资回收期：约 10 个月

需要强调的是，这是一套工业级完整改造的测算；我们当时做的只是桌面原型验证，距离这些指标还有很长的工程化路要走。

9.3 落地路径与风险

报告建议分三阶段推进：

  1. 阶段一：试点部署（1–2 个月） —— 选择 1–2 台设备做改造试点，验证技术可行性与实际效果；
  2. 阶段二：功能完善（2–3 个月） —— 多工位协同、人机协作界面、工艺数据库建设；
  3. 阶段三：全面推广（3–6 个月） —— 生产线全面智能化改造、员工培训、系统优化。

主要风险与应对：

设备兼容 现有产线改造适配困难 模块化设计、渐进式改造

9.4 行业认可

在方案初稿与回访过程中，我们得到了信源管理层与技术人员的高度认可，包括项目部负责人、技术员工和 IT 负责人，为方案的落地可行性提供了宝贵的现场验证。

10 总结与展望

10.1 学到的东西

  • 模型部署不是简单转换：ONNX → RKNN 的过程中，算子兼容、输入格式、后处理对齐都会踩坑，需要反复验证输出一致性；
  • 视觉与控制的耦合很关键：像素坐标到电机步数的标定误差会直接体现在补喷精度上，标定和数据同步必须一起做；
  • 原型要快速闭环：在红鸟挑战营这种短期项目里，先把端到端跑通，再逐步优化单点指标，比一开始追求每个模块完美更重要。

10.2 后续可优化方向

  • 引入深度相机或结构光：解决 2D 相机对复杂沟槽深度信息缺失的问题；
  • 轨迹规划算法优化：当前是点到点运动，可以加入连续轨迹和速度前瞻，提升节拍；
  • 多工件型号自适应：通过模板匹配或 few-shot 学习，让系统能自动适应新工件；
  • 工业现场验证：在真实粉末涂装产线上做长时间运行，暴露粉尘、振动、温漂等问题。

作为团队项目的一部分，我主要负责端侧部署与运动控制；也正是这次跨角色的合作让我第一次完整走通了「算法训练 → 边缘部署 → 硬件控制 → 系统集成」的链路，虽然只是一套原型，但对我理解机器人与 AI 在工业场景中的落地非常有帮助。
